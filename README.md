# SkillBridge — Skill Gap Analysis & Personalized Career Roadmap

A full-stack prototype that helps students understand their current skills, industry-required skills for a chosen role, the gap between them, and an AI-generated learning roadmap to become job-ready.

## Tech Stack

- **Frontend:** React (Vite), Tailwind CSS, Lenis, Framer Motion, Recharts. Hero uses Spline 3D (with CSS fallback if the scene fails to load).
- **Backend:** Node.js, Express.js, MongoDB (Mongoose)
- **AI:** OpenAI API (with mock fallback when key is missing)

## Prerequisites

- Node.js 18+
- MongoDB running locally (or use MongoDB Atlas and set `MONGODB_URI`)

## Setup

### 1. Backend

```bash
cd backend
cp .env.example .env
# Edit .env: set MONGODB_URI if needed, add OPENAI_API_KEY for AI roadmap (optional)
npm install
npm run dev
```

Backend runs at **http://localhost:5000**.

### 2. Frontend

```bash
cd frontend
cp .env.example .env
# Optional: set VITE_API_URL if backend is not on http://localhost:5000
npm install
npm run dev
```

Frontend runs at **http://localhost:5173**.

### 3. MongoDB

- **Local:** Start MongoDB and use `MONGODB_URI=mongodb://localhost:27017/skill-gap-platform` in `.env`.
- **Atlas:** Create a cluster, get connection string, and set `MONGODB_URI` in `.env`.

## Environment Variables

### Backend (`.env`)

| Variable         | Description                    | Default                          |
|------------------|--------------------------------|----------------------------------|
| `PORT`           | Server port                    | `5000`                           |
| `MONGODB_URI`    | MongoDB connection string     | `mongodb://localhost:27017/...`  |

| Variable        | Description              | Default                |
|-----------------|--------------------------|------------------------|

## API

- **POST /api/user/assessment**  
  Body: `{ name?, skills[], careerGoal }`  
  Returns: `{ userId, skillGap }` (userSkills, requiredSkills, matchedSkills, missingSkills, matchPercentage).

- **POST /api/roadmap/generate**  
  Body: `{ userSkills[], careerGoal, missingSkills[] }`  
  Returns: `{ weeks: [{ week, title, topics[], practice }], generatedWithMock? }`.

- **GET /api/health** — Returns `{ status: 'ok' }` if the server is up.
- **GET /api/user/recent?limit=10** — Returns recent assessments from the database (to verify storage).
- **GET /api/job-roles** — Returns all job roles and required skills (to verify seed data).

## Verifying backend and database

1. **Check backend is running**  
   Open in browser or use curl:
   ```bash
   curl http://localhost:5000/api/health
   ```
   You should see: `{"status":"ok"}`.

2. **Check seed data (job roles)**  
   Open in browser: **http://localhost:5000/api/job-roles**  
   You should see a list of 4 job roles (Software Developer, Data Analyst, ML Engineer, Web Developer) with their `requiredSkills`.

3. **Check that assessments are stored**  
   - Submit at least one assessment from the app (frontend).  
   - Then open: **http://localhost:5000/api/user/recent**  
   You should see your assessment(s) with `name`, `skills`, `careerGoal`, and `createdAt`.

4. **Using MongoDB directly**  
   - **MongoDB Compass:** Connect to `mongodb://localhost:27017`, open database `skill-gap-platform`, collections `users` and `jobroles`.  
   - **Shell:** `mongosh` then `use skill-gap-platform`, `db.users.find()`, `db.jobroles.find()`.

## Flow

1. **Landing** → “Start Skill Assessment”
2. **Assessment** → Select skills + career goal → Submit
3. **Dashboard** → View skill gap (charts + lists) → “Generate my roadmap”
4. **Roadmap** → AI (or mock) weekly learning plan
